# MNIST Classification – Classical ML vs CNN

This project explores different machine learning approaches for digit classification on the MNIST dataset.  

The project was developed collaboratively, with different models implemented by different team members, and the best-performing classical model was later compared to CNN architectures.

---

# Project Structure

## notebooks/

Contains all experiments and model training notebooks.

- **00_process_data.ipynb**  
  Loads the MNIST dataset, normalizes the images, splits the data into training, validation and test sets, and saves them locally in a `processed/` folder. 

- **01_leyla.louache.ipynb** 

- **02_Tetic-Seiwerth.Lucas.ipynb**  

- **03_Ghadiri.Seyedeh.Sara.ipynb** 

- **04_Eltantawy.Mohamed.ipynb**  

---

## models/

This folder contains pre-trained **CNN models** used for comparison with the ensemble models developed in this project.

The CNN models were trained during the course and exported as `.keras` files. They are directly evaluated on the test dataset in the notebook `notebooks/01_leyla.louache.ipynb`.

---

## results/

Contains `.json` files generated by notebooks **01–04**.  
These files store the evaluation metrics of the different models.

The notebook `final_comparison.ipynb`, located in this folder, loads these results and provides the final comparison between the selected ensemble models and the CNN models.


---

## plot/

Contains the figures generated during the experiments. All plots produced in the notebooks are saved in this folder.

---

# Workflow

To reproduce the experiments:

1. Run `notebooks/00_process_data.ipynb`

This will create the local `processed/` folder containing the normalized train/validation/test datasets.

2. Run the classical and ensemble models in the `notebooks/` folder:

   - `01_leyla.louache.ipynb`
   - `02_Tetic-Seiwerth.Lucas.ipynb`
   - `03_Ghadiri.Seyedeh.Sara.ipynb`
   - `04_Eltantawy.Mohamed.ipynb`

3. Compare the best classical model with CNN models in the `results/` folder:
   - `final_comparison.ipynb`

All evaluation results are already stored in the `results/` folder.  The comparison can therefore be reproduced directly by running `results/final_comparison.ipynb`, without rerunning the previous experiments.
