# MNIST Classification – Classical ML vs CNN

This project explores different machine learning approaches for digit classification on the MNIST dataset.  

The project was developed collaboratively, with different models implemented by different team members, and the best-performing classical model was later compared to CNN architectures.

---

# Project Structure

## notebooks/

Contains all experiments and model training notebooks.

- **00_process_data.ipynb**  
  Loads the MNIST dataset, normalizes the images, splits the data into training, validation and test sets, and saves them locally in a `processed/` folder.

  The `processed/` folder is created locally and ignored by GitHub, so this notebook must be executed first before running the others.

- **01_decision_tree.ipynb** *(Eltantawy Mohamed)*  
  Training and evaluation of a Decision Tree classifier.

- **02_knn.ipynb** *(Tetic-Seiwerth Lucas)*  
  Training and evaluation of a K-Nearest Neighbors classifier.

- **03_logistic_regression.ipynb** *(Ghadiri Seyedeh Sara)*  
  Logistic Regression model training and evaluation.

- **04_ensemble.ipynb** *(Louache Leyla)*  
  Ensemble learning experiments combining multiple classical models.  

- **05_robustness_evaluation.ipynb**   
  Compares the best classical model with the CNN models seen during the course.  
  Evaluation is performed on:
  - clean test images
  - shifted images
  - rotated images
  - mixed perturbations

---

## models/

Contains pre-trained **CNN models** used for comparison in the robustness evaluation notebook.

These models were trained during the course and exported as `.keras` files.

---

## results/

Contains `.json` files generated by notebooks **01–04**.  
These files store the evaluation metrics of the different models, allowing comparison between approaches and selection of the best classical model.

---

# Workflow

To reproduce the experiments:

1. Run notebooks/00_process_data.ipynb

This will create the local `processed/` folder containing the normalized train/validation/test datasets.

2. Train and evaluate the classical models:

01_decision_tree.ipynb
02_knn.ipynb
03_logistic_regression.ipynb
04_ensemble.ipynb

3. Compare the best classical model with CNN models:

05_robustness_evaluation.ipynb
